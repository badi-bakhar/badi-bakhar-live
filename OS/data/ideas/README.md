# data/ideas/

Original ideas, insights, and thoughts — not sourced from external content. These are your own thinking.

## What belongs here

- A realization you had in the shower
- A connection you noticed between two unrelated things
- An opinion you've been developing over time
- An interpretation of something you read or experienced

If the note is *primarily* a summary of someone else's work, it belongs in `web/`, `youtube/`, etc. If it is *primarily your own thinking* — sparked by whatever trigger — it belongs here.

## File Naming Convention

```
YYYY-MM-DD-slug.md
```

Example: `2026-03-17-small-wins-compound-like-interest.md`

## Frontmatter

```yaml
---
title:
date: YYYY-MM-DD
source: manual
tags: []
---
```

## Atomic Notes

Follow the Zettelkasten principle: **one idea per file**. Keep each note focused on a single, standalone claim or insight. This makes linking and retrieval much more effective over time.

A good idea note can usually be expressed in one or two sentences. The rest of the note is elaboration and links.
