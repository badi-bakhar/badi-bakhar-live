# data/instagram/

Instagram-sourced knowledge — reels, posts, carousels, and educational content.

## File Naming Convention

```
YYYY-MM-DD-slug.md
```

Example: `2026-03-17-ankur-warikoo-decision-framework.md`

## Frontmatter

```yaml
---
title:
date: YYYY-MM-DD
source: instagram
url:
creator:
content_type: reel | post | carousel | story
tags: []
summary:
---
```

- `creator` is the Instagram handle (e.g. `@ankurwarikoo`).
- `content_type` describes the format of the original post.
- `url` should point to the specific post if you have the link; otherwise omit it.

## Notes

Instagram content is often visual — carousels and reels in particular. When taking notes, transcribe or paraphrase the key points rather than just saving a link. Links rot; your notes should stand on their own.

For carousels, a brief numbered list of each slide's key idea works well.
