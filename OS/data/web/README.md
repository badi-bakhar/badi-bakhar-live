# data/web/

Web-sourced knowledge — articles, blog posts, and online reads captured from the internet.

## File Naming Convention

```
YYYY-MM-DD-slug.md
```

Example: `2026-03-17-why-cities-grow.md`

The date should reflect when you captured or read the piece, not its original publication date.

## Frontmatter

```yaml
---
title:
date: YYYY-MM-DD
source: web
url:
author:
tags: []
summary:
---
```

- `url` is required — always link back to the original.
- `author` is optional but recommended for attribution.
- `summary` should be 1-3 sentences in your own words.

## Notes

Keep the note itself focused on what *you* found valuable — not a full reprint of the article. Quotes, key ideas, and your own reactions are preferred.
